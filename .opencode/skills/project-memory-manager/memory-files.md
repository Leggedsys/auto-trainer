# 记忆文件职责

本文件说明 `plans/` 中各记忆文件的职责边界。

## `plans/memory_policy.yaml`

作用：

- 记忆协议本体
- 定义记忆结构、保留规则、压缩规则、视角规则

特点：

- 它不是工作记录
- 它是记忆系统的约束说明

什么时候必须读：

- 开始复杂规划或复杂实现前
- 要更新记忆文件前
- 要判断哪些信息应保留、压缩或删除时

## `plans/PROJECT_MEMORY.md`

作用：

- 存放 `auto-trainer` 项目开发的长期稳定记忆

应该包含：

- 项目终局
- 不可违背约束
- 总体路线
- 当前阶段判断
- 当前最高优先级
- 少量关键决策

不应该包含：

- 过程日志
- 局部任务细节
- 可轻易重推的信息

## `plans/NEXT_ACTIONS.md`

作用：

- 存放当前真正影响开发顺序的行动信息

应该包含：

- 当前目标
- 当前最重要的几件事
- 当前明确不做的事
- 下一阶段进入条件

不应该包含：

- 已完成历史列表
- 大段长期规划解释

## `plans/worklog/inbox.md`

作用：

- 原始工作记录缓冲区
- 接收过程性、探索性、未压缩的内容

应该包含：

- 临时判断
- 候选决策
- 试错记录
- 未定型风险
- 尚不确定是否值得长期保留的观察

不应该包含：

- 伪装成长期记忆的稳定结论
- 同一事实的大量重复版本

## 关系总结

- 协议在 `memory_policy.yaml`
- 长期稳定记忆在 `PROJECT_MEMORY.md`
- 当前行动在 `NEXT_ACTIONS.md`
- 原始过程缓冲在 `worklog/inbox.md`

不要混用职责。
