# 压缩规则

本文件说明工作结束后，如何把原始记录压缩进稳定记忆或当前行动。

## 1. 先写哪里

工作过程中，以下内容先写入 `plans/worklog/inbox.md`：

- 临时判断
- 候选方案
- 局部风险
- 探索性发现
- 试错结论

不要一出现想法就写进长期记忆。

## 2. 什么时候进入 `PROJECT_MEMORY.md`

满足以下至少两项时，才考虑进入长期记忆：

1. 之后难以从代码和现状重新推出
2. 会改变未来决策
3. 已经是最高层、最稳定的表达

适合进入长期记忆的内容：

- 终局定义变化
- 路线变化
- 能力边界变化
- 优先级变化
- 不可逆约束
- 关键决策

## 3. 什么时候进入 `NEXT_ACTIONS.md`

当信息会改变当前开发顺序时，写入 `NEXT_ACTIONS.md`。

适合进入当前行动的内容：

- 当前目标变化
- 当前最重要任务变化
- 当前明确不做的事变化
- 下一阶段进入条件变化

## 4. 什么时候应删除

以下内容通常应从 worklog 删除，而不是保留：

- 纯过程噪音
- 已解决线程
- 重复判断
- 仅为完整性保留的过程描述
- 能从现有记忆和代码直接重推的说明

## 5. 压缩时机

以下情况应触发压缩：

- 一项复杂工作结束
- 用户明确要求整理或压缩
- `plans/worklog/inbox.md` 变长且开始重复
- 某个讨论线程已经收敛

## 6. 压缩目标

压缩不是“写一份更短的日志”，而是：

- 决策性内容进入长期记忆
- 当前执行内容进入当前行动
- 过程噪音被删除
- worklog 只留下未解决但短期仍有价值的碎片
